# AGENTS.md

本文件适用于仓库根目录及全部子目录。自动化代理、开发者和审阅者修改本仓库时必须遵守以下约束。

## 1. 项目边界

- 本仓库负责领星 ERP 本地产品、分类、SPU 和自定义字段的同步与安全写回。
- 不在本仓库内修改销量预测或面料预测业务逻辑；跨仓库改造必须单独立项、单独 PR。
- 颜色体系字段权威值域为：`A2023`、`B2024`、`待定`。当前存量治理脚本只自动写入经过审阅的 `A2023`。

## 2. 凭据与数据安全

- 不提交 `.env`、access token、app secret、数据库密码或生产导出文件。
- 示例中使用占位符，不复制生产凭据。
- 不在日志、PR 描述和测试数据中暴露真实密钥。

## 3. Git 工作流

- 基于最新 `main` 创建功能分支；不要直接向 `main` 推送业务变更。
- 一个 PR 只处理一个明确主题，正文必须写清风险、验证方式和未验证事项。
- 不用 `git add .` 无差别提交运行产物；提交前检查变更文件清单。
- Linux 服务器的文件权限变化必须有明确业务理由；禁止让 `core.fileMode` 产生的假修改混入提交。

## 4. 领星写回硬约束

- 批量写回必须默认 dry-run；正式写入必须显式使用 `--apply` 或 `--confirm`，并使用人工审阅后的范围文件。
- 所有 `product/set` 写入必须复用 `lx_product_m/services/product_write_guard.py`：
  - 写前重新读取实时产品详情；
  - 使用实时 `product_name`；
  - 合并完整实时自定义字段；
  - 非分类变更保留当前分类；
  - 实时详情缺失或品名为空时禁止写入。
- 写入成功不能只依据接口 `code=0`。批量脚本必须写后回读，至少复查 SKU、品名、分类、目标字段和请求中携带的全部原有自定义字段。
- 写后复查不一致必须计为失败、输出可追溯失败清单并返回非 0 退出码。
- `code=103` 使用固定 30 秒等待，最多重试 3 次；最终失败不得中断其他 SKU，但必须记录错误码、消息和 request_id。
- 颜色体系批量 `--apply` 默认只允许北京时间 `00:00-06:00`。解除时间保护必须人工确认没有并行写入任务。

## 5. 规则与范围治理

- 第一轮正式写入只处理审阅文件中的 `A2023`；`待定` 不调用写接口。
- 第二轮只能从第一轮审阅文件的待定范围进入，保留数量断言，避免范围漂移。
- 开发年份仅接受清洗后的四位年份：`2000-2024` 可判 A2023，`2025-2100` 维持待定；其他数字和文本列为意外值。
- SPU 级属性不能替代 SKU 级证据用于判断活跃新增颜色；使用此类规则时必须保留业务前提和追溯名单。

## 6. 验证要求

提交前至少执行：

```bash
python -m compileall -q lx_product_m scripts tests
python -m unittest discover -s tests -v
```

涉及生产接口的变更还必须：

1. 先做单 SKU 写入；
2. 写后实时回读；
3. 核对品名、分类和其他自定义字段无变化；
4. 再执行批量任务。

无法连接生产数据库或领星 API 时，必须在 PR 中明确写“未执行生产验证”，不得暗示已经验证。
