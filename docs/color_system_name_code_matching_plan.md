# 颜色体系：SKU 颜色代码 + 品名颜色匹配方案

## 1. 目标与边界

- 主分析范围：产品快照中“颜色体系”为空的全部 SKU（范围 A）。
- 第一阶段仅生成只读诊断 Excel，不调用领星 `product/set`。
- 已有 A2023/B2024 或其他非空值只用于验证和冲突审计，不覆盖。
- 无销量且无 Listing 的老旧 SKU 排在最低处理优先级，但仍保留在全量分析结果中。
- 映射文件属于业务数据，不提交到公开仓库；运行时从服务器本地路径读取。

## 2. 已确认的 SKU 结构

1. 常规：`款号-颜色代码-尺码`
2. 少量：`款号-颜色代码尺码`
3. 少量：`款号-SHORT-颜色代码-尺码`（同样兼容 LONG/TALL/PETITE）
4. 多色套装：含 `PCS` 标记，如：
   - `NKW033-5PCS-WH-BE-SD-GY-MGR-XL`
5. 款号本身不含连字符。
6. 颜色代码不含连字符；解析以颜色映射表中的实际代码集合为准，不只依赖固定位置。

示例：

- `KZ291-SHORT-BK-S` -> 款号 KZ291、版型 SHORT、颜色 BK、尺码 S。
- `NY006-SN-40DD` -> 款号 NY006、颜色 SN、尺码 40DD。
- `NKW033-5PCS-WH-BE-SD-GY-MGR-XL` -> 5 件套、多颜色 WH/BE/SD/GY/MGR、尺码 XL。

## 3. 映射表处理规则

输入列：序号、缩写、全拼、中文、Pantone 色号、颜色体系。

- 按 Excel 原始行序读取。
- 同一体系内同一代码重复时：第一条为“主映射”。
- 后续重复行保留为“次级历史映射”，不得覆盖主映射。
- 主映射精确命中可进入高置信度；只命中次级映射时进入中置信度/人工审阅。
- 同一代码同时存在于 A2023 与 B2024 时，必须结合品名颜色判断。
- 同一代码和同一颜色同时存在于两套体系时，代码+品名仍无法区分，保持待定。

当前映射表基线统计：

- 共 393 行：B2024 199 行，A2023 194 行。
- B2024：199 个唯一代码，无同体系重复代码。
- A2023：177 个唯一代码，17 个代码存在重复行；其中 14 个代码对应不同中文颜色。
- A2023/B2024 共有 58 个重叠代码；按主映射比较，5 个代码中文颜色相同、53 个不同。

## 4. 品名颜色标准化

- 清理空格、全角/半角标点和大小写差异。
- 颜色名末尾“色”可选：`赭` 与 `赭色`视为同一标准候选。
- 使用最长词优先，避免 `藏蓝色`被截成`蓝色`、`酒红色`被截成`红色`。
- 以下词不视为具体颜色：净色、纯色、撞色、拼色、花色、多色、有色、无色。
- 第一版不做开放式语义猜测；模糊错字只生成候选，不自动写入。

## 5. SKU 解析策略

### 5.1 常规与 SHORT/LONG

- 第一段固定作为款号。
- 跳过 SHORT/LONG/TALL/PETITE 等版型词。
- 颜色段必须能在映射代码集合中找到。
- 尺码允许包含数字（例如 40DD）；颜色代码是否有效由映射表决定。

### 5.2 颜色和尺码粘连

对于 `款号-颜色代码尺码`：

- 在第二段中按“最长颜色代码前缀”匹配。
- 剩余部分作为尺码。
- 若存在多个同长度可行代码，标记 SKU 结构歧义，不自动判定。

### 5.3 多色套装

- 识别包含 `PCS` 的段（大小写不敏感）。
- PCS 后至末尾尺码前的所有有效颜色代码作为颜色集合。
- 若所有颜色代码仅属于同一体系，可形成体系证据。
- 若颜色代码跨体系、存在未知代码或品名颜色数量不足，进入套装审阅，不自动写入。

## 6. 判定层级

### 高置信度候选

1. 当前标签为空。
2. SKU 颜色代码解析唯一。
3. 代码 + 品名颜色精确命中某一体系主映射。
4. 另一体系不匹配。

### 中置信度候选

- 代码只存在于一个体系，但品名颜色缺失或未命中。
- 品名只命中同体系的次级历史映射。
- 已确认别名命中。
- 多色套装全部代码指向同一体系，但品名不能完整验证。

中置信度只进入审阅，不直接写入。

### 待定或冲突

- A/B 主映射同时匹配。
- 同一代码和颜色在 A/B 完全相同。
- 代码与品名颜色均不一致。
- 无法解析颜色代码。
- 多个疑似颜色代码且无法确定结构。
- 多色套装包含跨体系或未知代码。
- 当前已有标签与推断结果不一致。

## 7. 处理优先级

自动检测数据库中所有可用订单年度表，并以可获得的完整历史计算销量：

- P1：有销量的 SKU。
- P2：无销量但仍有 Listing 的 SKU。
- P3：无销量且无 Listing 的 SKU（历史老旧 SKU，最低优先级）。

优先级只影响审阅和执行顺序，不改变颜色体系判定本身。

## 8. 无人工真值样本时的验证

- 使用当前已经明确写入的 A2023 SKU 作为 A2023 回测集，测量解析成功率、命中率和冲突率。
- B2024 暂无正式真值：Listing 时间晚于 2024-06-30、开发年份为 2025+ 只能作为辅助对照，不作为最终正确答案。
- 第一版诊断后，从高置信度 B2024 候选中人工抽查至少 100 条，确认准确率后再开发 B2024 写入能力。

## 9. 第一版输出 Excel

1. `汇总`
2. `全量判定明细`
3. `高置信度_A2023`
4. `高置信度_B2024`
5. `中置信度待审阅`
6. `冲突清单`
7. `多色套装`
8. `SKU结构异常`
9. `映射表质量`
10. `A2023回测结果`

关键字段：SKU、品名、SPU、解析结构、颜色代码、品名颜色、A/B 主映射、A/B 次级映射、拟打体系、置信度、判定原因、销量、Listing 状态、处理优先级、是否允许自动写入。

## 10. 正式写入前置条件

- 只允许从人工审阅后的 Excel 写入。
- 第一版代码不包含 `--apply`。
- 后续写入脚本仅接受 A2023/B2024；待定、冲突不得写。
- 写前实时读取；当前标签为空才允许写。
- 当前标签相同则跳过，不同则冲突退出。
- 写前完整快照备份、保留全部字段、写后实时复查、失败清单可重跑。
- 按单条 -> 100 条 -> 1000 条 -> 全量逐级验证。
